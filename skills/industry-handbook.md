# 产业认知手册生成

按照标准三阶段流程（信息搜集 → 数据分析 → 报告撰写），为一个产业生成 5 份深度报告，再整合为多标签 HTML 手册。

## 目录结构

```
outputs/industry-researcher/
├── {产业名}/
│   ├── 报告1_xxx.md / .html
│   ├── 报告2_xxx.md / .html
│   ├── ...
│   └── 整合报告/
│       └── 2026{产业名}产业认知手册.html   ← 5标签页最终产物
└── 分享版/
    ├── 2026汽车产业认知手册.html
    └── 2026半导体产业认知手册.html
```

## 三阶段流程

### 第一阶段：信息搜集
- 确定产业范围，列出 5 个关键维度（行业全景、市场竞争、中国企业、垂直应用、趋势博弈）
- 每个维度收集公开数据（财报、研报、行业统计数据）
- 数据基准以最近完整年份 + 当前上半年为准

### 第二阶段：数据分析
- 整理产业链结构、市场份额、企业对比
- 提炼核心数字（市场规模、头部企业营收、增长率等）
- 识别关键趋势和博弈逻辑

### 第三阶段：报告撰写
- 每份报告 5-7 章，商业分析风格，面向小白读者
- Markdown 撰写 → `build_html.py` 转为 HTML
- 5 份报告合并为单文件手册，标签页切换

## 手册规范

### 设计
- 封面：标题 + 10 个必记数字卡片
- 5 个标签页，每页对应一份报告
- 支持暗色模式（`prefers-color-scheme: dark`）
- 底部互相跳转链接（同产业不同手册之间）

### 内容风格
- 商业分析风格，通俗易懂
- 避免学术引用格式和过于技术化的术语
- 避免偏激用语（如"卡脖子"→"出口管制"，"打压"→"限制"）
- 面向小白读者，配产业链图解、对比表格

### 可视化
- 图表使用 `dataviz` skill 生成
- 车标/logo 使用真实图片 base64 内嵌
- 产业链用彩色流程图展示

## 构建脚本

| 脚本 | 用途 |
|------|------|
| `scripts/build_html.py` | Markdown → HTML（路径按产业前缀） |
| `scripts/merge_brand_history.py` | 注入品牌简史面板 |
| `scripts/build_semi_handbook.py` | 一键构建半导体5标签手册 |

**Python 环境：** conda env `auto-report` (Python 3.12, markdown, jinja2)

## 分享版

- 两个手册平级存放，互跳链接改为同级文件名
- 方便发给朋友在电脑上浏览
- 部署到 GitHub Pages：boltnye/industry-handbooks
